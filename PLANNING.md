> 최종 수정: 2026-06-02

## 1. 프로젝트 개요

### 배경

별을 보러 나갔다가 구름이 잔뜩 끼거나 보름달이 너무 밝아서 실망한 경험이 있다.  
"언제 가야 별이 잘 보일까?" 라는 질문에 답해주는 서비스가 없어서 직접 만들기로 했다.

### 핵심 인사이트

- **달 위상**이 별 관측의 가장 큰 변수다. 그믐(신월) 전후 ±2일이 월중 최적 기간.
- **날씨(운량)** 가 두 번째 변수. 맑은 날도 보름달이면 의미 없다.
- 두 변수를 합산한 **별 관측 지수**를 달력에 시각화하면 직관적으로 계획을 세울 수 있다.

### 목표

| 목표                         | 측정 기준                             |
| ---------------------------- | ------------------------------------- |
| 별 관측 최적일을 한눈에 확인 | 달력에서 3초 안에 이번 달 최적일 파악 |
| 관측 기록 보관               | 일기 + 사진으로 추억 저장             |
| 관측 습관 형성               | 통계 화면으로 "나는 몇 번 봤나" 확인  |

## 3. 기능 명세

### 3-1. 달력 (메인)

**목적**: 이번 달 전체를 보면서 "언제 가면 좋을까" 판단

| 항목        | 내용                                                         |
| ----------- | ------------------------------------------------------------ |
| URL         | `/` (쿼리: `?month=YYYY-MM`)                                 |
| 구성 요소   | 월 이동 버튼 · 요일 헤더 · 6주 × 7일 그리드                  |
| 날짜 셀     | 양력 일 · 음력 일 · 달 이모지 · 별 관측 지수 숫자(색상 코딩) |
| 오늘 강조   | 테두리 + 배경색                                              |
| 그믐 최적일 | 녹색 글로우 효과                                             |
| 날씨        | Geolocation → 현재 위치 기반, 동의 없으면 서울 기본값        |

**별 관측 지수 산출 공식**

```
달 위상 점수 (0~50):
  raw = (cos(phase / 29.53 × 2π) + 1) / 2 × 100
  moonScore = round(raw / 2)  -- 0~50으로 스케일 다운

날씨 점수 (0~50):
  cloudScore  = (1 - cloudCover / 100) × 35     -- 0~35
  precipScore = (1 - precipProb / 100) × 15     -- 0~15
  weatherScore = clamp(cloudScore + precipScore, 0, 50)

총점 = moonScore + weatherScore  (0~100)
```

**등급 색상 코딩**

| 등급 | 점수 | 색상             |
| ---- | ---- | ---------------- |
| 최상 | 80+  | 초록 (`#4ade80`) |
| 좋음 | 60+  | 청록 (`#22d3ee`) |
| 보통 | 40+  | 노랑 (`#facc15`) |
| 나쁨 | 20+  | 주황 (`#fb923c`) |
| 최악 | 0+   | 빨강 (`#f87171`) |

### 3-2. 날짜 상세

**목적**: 특정 날 클릭 시 상세 정보 확인

| 항목         | 내용                                                     |
| ------------ | -------------------------------------------------------- |
| URL          | `/day/YYYY-MM-DD`                                        |
| 달 정보      | 이모지(크게) · 위상명 · 음력 날짜 · 조도 %               |
| 별 관측 지수 | 숫자 + 등급 + 게이지 바 + 달/날씨 점수 분해              |
| 날씨         | 운량 % · 날씨 설명 · 강수확률 (오늘: 실시간, 미래: 예보) |
| 날씨 제한    | Open-Meteo 16일 이내만 예보 제공, 초과 시 안내 문구      |
| 그믐 최적일  | 배지 표시                                                |

### 3-3. 일기

**목적**: 관측 후 기억을 텍스트 + 사진으로 남김

| 항목           | 내용                                                                           |
| -------------- | ------------------------------------------------------------------------------ |
| URL            | `/diary` (목록) · `/diary/YYYY-MM-DD` (상세) · `/diary/YYYY-MM-DD/edit` (수정) |
| 접근 제한      | 로그인 필수 (미인증 → `/login` 리다이렉트)                                     |
| 작성 항목      | 날짜 · 본문(자유) · 사진(선택)                                                 |
| 자동 저장 항목 | 달 위상 · 달 이모지 · 운량 (작성 시 자동 삽입)                                 |
| 목록           | 날짜 역순 · 썸네일(사진 없으면 달 이모지) · 첫 줄 미리보기                     |
| 수정/삭제      | 상세 페이지에서 가능                                                           |
| 사진 저장      | Supabase Storage `diary-photos` 버킷                                           |

---

### 3-4. 관측 체크

**목적**: "오늘 별 봤어요" 한 번의 탭으로 기록

| 항목       | 내용                                                               |
| ---------- | ------------------------------------------------------------------ |
| 위치       | 날짜 상세 페이지 또는 달력 날짜 셀                                 |
| 동작       | 토글 — 체크 시 `observations` 테이블에 INSERT, 다시 클릭 시 DELETE |
| 날짜당 1회 | `(user_id, date)` 유니크 제약                                      |
| 위치 저장  | Geolocation 동의 시 lat/lng 함께 저장                              |

### 3-5. 통계

**목적**: "내가 얼마나 자주 별을 봤나" 시각화

| 항목        | 내용                                          |
| ----------- | --------------------------------------------- |
| URL         | `/stats`                                      |
| 요약 카드   | 총 관측 횟수 · 올해 가장 많이 관측한 달       |
| 월별 차트   | 올해 1~12월 막대 차트 (Recharts BarChart)     |
| 연간 히트맵 | GitHub 잔디 스타일, 관측한 날 강조 (CSS Grid) |
| 접근 제한   | 로그인 필수                                   |

## 4. 화면 흐름 (User Flow)

```
[첫 방문]
  → 달 떠오르는 2초 로딩 애니메이션
  → 메인 달력 (오늘 요약 카드 + 이번 달 달력)
     ├─ 날짜 클릭 → /day/YYYY-MM-DD (상세)
     │     └─ "별 봤어요" 체크 (로그인 필요)
     ├─ 일기 탭 → /diary (로그인 필요)
     │     ├─ 새 일기 작성
     │     └─ 일기 상세/수정/삭제
     └─ 통계 탭 → /stats (로그인 필요)

[미로그인 접근]
  → /login (카카오/구글 버튼)
  → OAuth 인증 완료
  → 원래 페이지로 돌아옴
```

## 5. 기술 아키텍처

### 렌더링 전략

| 페이지                     | 전략             | 이유                                         |
| -------------------------- | ---------------- | -------------------------------------------- |
| `app/page.tsx` (메인 달력) | Client Component | Geolocation, 로딩 애니메이션                 |
| `app/day/[date]/page.tsx`  | Server Component | SEO + 날씨 fetch 서버에서                    |
| `app/diary/page.tsx`       | Server Component | 인증 체크 + DB 조회 서버에서                 |
| `app/stats/page.tsx`       | Server Component | 통계 계산 서버에서, StatsCharts만 클라이언트 |
| `components/MoonCalendar`  | Client Component | 월 이동 상태, 날씨 fetch                     |

### API 구조

| 엔드포인트                   | 메서드              | 설명                          |
| ---------------------------- | ------------------- | ----------------------------- |
| `/api/lunar?date=YYYY-MM-DD` | GET                 | 음력 변환 결과 반환           |
| `/api/diary`                 | GET · POST          | 일기 목록 조회 / 새 일기 작성 |
| `/api/diary/[date]`          | GET · PUT · DELETE  | 일기 상세 / 수정 / 삭제       |
| `/api/observation`           | GET · POST · DELETE | 관측 체크 조회 / 추가 / 취소  |
| `/api/observation/check`     | GET                 | 특정 날짜 관측 여부 확인      |
| `/api/auth/[...nextauth]`    | GET · POST          | NextAuth OAuth 핸들러         |

### 날씨 데이터 캐싱

| 용도               | 캐시 전략                                      |
| ------------------ | ---------------------------------------------- |
| 현재 날씨 (실시간) | `next: { revalidate: 600 }` — 10분             |
| 월간 예보          | `next: { revalidate: 3600 }` — 1시간           |
| 음력 변환          | `cache: "force-cache"` — 영구 (계산 결과 불변) |

### Supabase 클라이언트 분리

```
createBrowserClient()  — anon key, RLS 적용, 클라이언트 컴포넌트용
createServerClient()   — service_role key, RLS 우회, API Routes/Server Components용
```

## 6. 데이터베이스 설계

### ERD 요약

```
users (1) ──< diaries (N)       : user_id FK
users (1) ──< observations (N)  : user_id FK
```

### 테이블 정의

**users**
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | text PK | NextAuth user.id (OAuth sub) |
| email | text UNIQUE | 이메일 |
| name | text | 이름 |
| image | text | 프로필 이미지 URL |
| provider | text | `'kakao'` 또는 `'google'` |
| created_at | timestamptz | |

**diaries**
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid PK | auto |
| user_id | text FK | → users.id |
| date | date UNIQUE/user | 관측 날짜 |
| content | text | 일기 본문 |
| photo_url | text | Supabase Storage URL |
| moon_phase | text | 위상 영문 키 |
| moon_emoji | text | 달 이모지 |
| weather_code | integer | WMO 날씨 코드 |
| cloud_cover | integer | 운량 0~100 |
| created_at / updated_at | timestamptz | |

**observations**
| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | uuid PK | auto |
| user_id | text FK | → users.id |
| date | date UNIQUE/user | 관측 날짜 |
| location_name | text | 지역명 (선택) |
| lat / lng | numeric(9,6) | 좌표 (선택) |
| created_at | timestamptz | |
